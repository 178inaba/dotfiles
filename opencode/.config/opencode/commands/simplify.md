---
description: 最近変更したファイルのコード品質・再利用性・効率性を確認して修正
---

最近変更したファイル（`git diff HEAD~1...HEAD --name-only` または未コミットの変更）を対象に、以下の観点でレビューして問題を修正してください。

$ARGUMENTS

## レビュー観点

- **コード再利用性**: 重複コード、共通化できる処理、既存ユーティリティの活用漏れ
- **コード品質**: 可読性、保守性、命名、不要な複雑さ
- **効率性**: 無駄な処理、パフォーマンス上の問題

## 実行手順

1. `git diff HEAD --name-only` と `git diff HEAD~1...HEAD --name-only` で変更ファイルを特定
2. 変更ファイルの内容を読み込む
3. 3つの観点（再利用性・品質・効率性）を並行してレビュー
4. 発見した問題を優先度順に整理
5. 修正を適用してコミット

## 注意事項

- 引数（`$ARGUMENTS`）が指定された場合は、その観点を優先してレビュー
- 引数がない場合は3つの観点すべてをバランスよく確認
- 動作を変える修正は行わない（リファクタリングのみ）
- 修正後はテスト・Lintを実行して品質を確認
