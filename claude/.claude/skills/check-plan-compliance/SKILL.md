# /check-plan-compliance

## 使用方法
```
/check-plan-compliance
```

## 実行内容

### 1. 参照文書の収集
1. プロジェクトルートの `CLAUDE.md` を読む（無ければ「対象なし」として終了）
2. CLAUDE.md から以下の `.md` リンクを抽出:
   - 標準Markdownリンク: `[text](path.md)` 形式
   - Claude `@import` 記法: `@path/to/file.md` または `@~/path/file.md`
3. 抽出したリンク先の `.md` ファイルを読む（プロジェクト内相対パス・ホーム絶対パス両対応）
4. リンク先文書から更にリンクされている `.md` があれば、1段だけ追跡して読む
5. リンク先が存在しない場合は警告のみ表示し続行

### 2. 計画との突合
収集した文書群と現計画を比較し、以下の観点で評価:
- **準拠**: 計画が文書のルール・パターンに沿っている箇所
- **乖離**: 計画が文書のルール・パターンに反している、または無視している箇所
- **CLAUDE.md側の更新候補**: 計画で採用する設計が文書に未記載で、汎用的に適用すべき場合

### 3. 報告フォーマット

```
## 準拠チェック結果

### 参照した文書
- <file1>
- <file2>

### 準拠している点
- <要約>

### 乖離・要検討点
- <文書のルール> vs <計画の内容> — <推奨対応>

### CLAUDE.md/関連文書の更新候補
- <候補。なければ「なし」>
```

## 前提条件
- 既に計画が存在する状態で呼ばれる（plan モード内、または計画ファイル参照時）
- プロジェクトルートで実行されること

## 注意事項
- このスキルは**事後検証用の安全網**であり、計画段階の思考に組み込むことが本来の姿
- issue-handle 経由では計画フェーズ冒頭で参照文書を自動読込するため、本スキルの単体呼び出しは「計画修正後の再確認」「issue-handle 外での自由な計画」での利用が中心
